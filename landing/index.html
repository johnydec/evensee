How it works

Three clicks to
authenticated access

No passwords. No login flows. No browser profiles. Just the headers you need.

1

Browse & click Capture

Open any site where you're logged in. Click the extension icon and hit Capture. The page auto-reloads to trigger fresh requests.

2

Headers captured

Auth headers appear instantly — Authorization, Cookie, CSRF tokens, session IDs. Auto-stops when done. Green check confirms it.

3

Copy & paste

One click copies ready-to-use JSON. Paste directly into your code, HTTP client, or hand it to an AI agent.

// Playwright await page.setExtraHTTPHeaders({ "Authorization": "Bearer eyJ...", "Cookie": "session=abc..." });

Features

Everything you need,
nothing you don't

Auto-capture

Listens for auth headers on any request. Auto-stops when captured. No manual selection needed.

One-click copy

Copy as JSON ready for Playwright, curl, or any HTTP client. No reformatting needed.

Multi-domain

Capture from multiple sites at once. Each session tracked separately with its own card.

Smart detection

Warns you about protected sites (Facebook, Google) where header replay won't work.

Auto-clear

Headers auto-delete after 5 minutes. Clipboard clears after 30 seconds. Configurable.

Tab-scoped

Only captures from the specific tab and domain you choose. No background monitoring.

Security

Paranoid by design

We handle your most sensitive data — auth tokens. Here's why you can trust us.

Memory only

Headers live in the service worker's memory. Never written to disk, localStorage, or any persistent storage. Close Chrome and they're gone.

Zero network calls

The extension makes exactly zero HTTP requests. No analytics, no telemetry, no external servers. Check the source code yourself.

Auto-destruct

Captured headers auto-delete after 5 minutes. Clipboard auto-clears after 30 seconds. Both configurable — or set to manual.
